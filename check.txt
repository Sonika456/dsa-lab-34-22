ЧЕК-ЛИСТ ТЕСТИРОВАНИЯ МОДУЛЕЙ: triangle_func.py И triangle_class.py

[ЧТО НУЖНО (ОБЩИЕ ТРЕБОВАНИЯ)]
- Программа должна принимать три аргумента (a, b, c), интерпретируемые как длины сторон треугольника.
- Аргументы должны быть числовыми типами (int или float).
- Каждая сторона должна быть строго больше нуля.
- Должно выполняться неравенство треугольника: сумма любых двух сторон строго больше третьей (a + b > c, a + c > b, b + c > a).
- При соблюдении всех условий программа должна корректно определить тип треугольника или создать объект класса.
- При нарушении условий программа должна завершить выполнение текущей операции и выбросить исключение IncorrectTriangleSides.

1. ПРАВИЛЬНОЕ ИСПОЛНЕНИЕ ПРОГРАММЫ (ПОЗИТИВНОЕ ТЕСТИРОВАНИЕ)
ЧТО ПЕРЕДАЁМ ВНУТРЬ:
- Наборы из трёх положительных чисел, полностью удовлетворяющих неравенству треугольника.
- Примеры: (3, 3, 3), (4, 4, 6), (3, 4, 5), (5.5, 5.5, 8.0)

ЧТО ОЖИДАЕМ:
- Функция get_triangle_type(): возврат строки с точным названием типа треугольника:
  * "equilateral" — если a == b == c
  * "isosceles" — если ровно две стороны равны
  * "nonequilateral" — если все стороны различны
- Класс Triangle(): успешное создание экземпляра без ошибок. Метод .triangle_type() возвращает аналогичные строки. Метод .perimeter() возвращает число (сумму сторон).
- Статус выполнения: Программа отработала без исключений, вернула корректный результат. Тест проходит (PASS).

2. НЕПРАВИЛЬНОЕ ИСПОЛНЕНИЕ ПРОГРАММЫ (НЕГАТИВНОЕ ТЕСТИРОВАНИЕ)
ЧТО ПЕРЕДАЁМ ВНУТРЬ:
- Наборы данных, нарушающие математические или типовые ограничения.
- Примеры нарушений:
  1. Нулевое значение: (0, 5, 5)
  2. Отрицательное значение: (-2, 4, 5)
  3. Вырожденный треугольник (сумма двух сторон равна третьей): (1, 2, 3)
  4. Невозможный треугольник (сумма двух сторон меньше третьей): (10, 2, 3)
  5. Некорректный тип данных: ("text", 4, 5), (None, 3, 3), ([1], 2, 3)

ЧТО ОЖИДАЕМ:
- Программа НЕ должна возвращать строку типа треугольника или создавать объект.
- Ожидается прерывание выполнения и генерация пользовательского исключения IncorrectTriangleSides.
- Статус выполнения: Программа корректно обработала ошибку, исключение поймано тестовым фреймворком. Если исключение не выбрасывается или программа падает с другой ошибкой — тест считается НЕПРОЙДЕННЫМ (FAIL).
